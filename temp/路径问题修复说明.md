# 图片路径 404 问题修复说明

## 🐛 问题诊断

**错误信息**:
```
Failed to load resource: the server responded with a status of 404 ()
images/backgrounds/ch01-church.png:1
```

**根本原因**:
- GitHub Pages 部署在子目录：`https://kidsmartghost.github.io/rift-oath/`
- 代码中使用绝对路径：`/images/backgrounds/ch01-church.png`
- 浏览器实际访问：`https://kidsmartghost.github.io/images/backgrounds/ch01-church.png` ❌
- 正确地址应该是：`https://kidsmartghost.github.io/rift-oath/images/backgrounds/ch01-church.png` ✅

---

## ✅ 解决方案

### 修改内容

在 `src/data/assets.js` 中添加基础路径配置：

```javascript
// 基础路径 - 使用 Vite 的 BASE_URL 或硬编码
const BASE_PATH = import.meta.env.BASE_URL || '/rift-oath/'

// 辅助函数：构建图片路径
function imagePath(path) {
  // 去掉开头的 ./ 或 /，然后加上 BASE_PATH
  const cleanPath = path.replace(/^[/\.]+/, '')
  return BASE_PATH + cleanPath
}

// 使用示例
backgrounds: {
  'ch01-church': {
    id: 'ch01-church',
    world: 'world1',
    image: imagePath('images/backgrounds/ch01-church.png'),  // ✅ 正确
    name: '圣罗兰教堂'
  }
}
```

### 构建效果

构建后的代码（`dist/assets/index-*.js`）：


## 📝 经验总结

### 问题根源

在 Vite 项目中，如果部署到子目录（如 GitHub Pages）：

❌ **错误做法**:
```javascript
// 绝对路径，会从域名根开始
image: '/images/backgrounds/ch01-church.png'
```

✅ **正确做法**:
```javascript
// 使用 Vite 的 BASE_URL
const BASE_PATH = import.meta.env.BASE_URL || '/rift-oath/'
image: BASE_PATH + 'images/backgrounds/ch01-church.png'

// 或者使用相对路径（Vite 会自动处理）
image: new URL('../images/backgrounds/ch01-church.png', import.meta.url).href
```

### Vite 配置

确保 `vite.config.js` 中设置了正确的 `base`：

```javascript
export default defineConfig({
  base: '/rift-oath/',  // GitHub Pages 子目录
  // ... 其他配置
})
```

### 最佳实践

1. **动态路径使用函数处理** - 不要硬编码路径
2. **使用 Vite 的环境变量** - `import.meta.env.BASE_URL`
3. **构建后验证** - 检查 `dist/` 目录中的实际路径
4. **部署前本地测试** - `npm run preview`
