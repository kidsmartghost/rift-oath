# 🔍 场景跳转问题诊断

## 问题现象
玩家在某些章节选择选项后，会跳回第一章开头（`scene_001`）

## 根本原因

### 1. 缺失的场景引用

检查发现 **56 个选项指向了不存在的场景**：

**第 3 章缺失的结局**:
- `ending_destroyer` (毁灭者结局)
- `ending_sacrifice` (牺牲结局)
- `ending_guardian` (守护者结局)
- `ending_tyant` (暴君结局)
- `ending_liberator` (解放者结局)
- `ending_true_rebirth` (真结局：重生)
- `ending_true_cycle` (真结局：轮回)

**第 4 章缺失的结局**:
- 同上，引用了第 3 章的结局场景

**第 5 章缺失的结局**:
- `ending_final_destroy` (最终结局：毁灭)
- `ending_final_godkill` (最终结局：弑神)
- `ending_final_sacrifice` (最终结局：牺牲)
- `ending_final_sleep` (最终结局：长眠)
- `ending_final_letgo` (最终结局：放手)
- `ending_final_merge` (最终结局：融合)
- `ending_final_guardian` (最终结局：守护者)
- `ending_final_free` (最终结局：自由)
- `ending_final_reincarnate` (最终结局：轮回)
- `ending_final_newguardian` (最终结局：新守护者)
- `ending_final_future` (最终结局：未来)
- `ending_final_blessing` (最终结局：祝福)
- `ending_final_cycle` (最终结局：轮回)

### 2. 错误的回退逻辑

```javascript
// src/pages/Game.vue line ~405
if (!scene) {
  console.error('场景不存在:', sceneId)
  // ❌ 回退到第一个场景
  const firstScene = getLoadedScript()?.scenes?.[0]
  if (firstScene) {
    loadScene(firstScene.id)  // 这会跳转到 scene_001
  }
  return
}
```

当玩家在第 5 章选择了一个结局选项（如 `ending_final_guardian`），但：
1. 第 5 章剧本中没有这个场景定义
2. 代码尝试加载 `ending_final_guardian`
3. 找不到场景，触发回退逻辑
4. 回退到第 5 章的第一个场景（或当前加载剧本的第一个场景）
5. 玩家看到自己回到了开头

---

## 解决方案

### 方案 1: 添加缺失的结局场景（推荐）

在对应的章节文件中添加所有引用的结局场景：

**在第 3 章添加**:
```json
{
  "id": "ending_destroyer",
  "worldId": "world3",
  "type": "ending",
  "title": "第三章结局：毁灭者",
  "dialogue": "你选择了毁灭...\n\n【第三章 毁灭者结局】",
  "background": "ch03-godfall",
  "characters": [],
  "nextSceneId": null
}
```

**在第 5 章添加所有最终结局**:
```json
{
  "id": "ending_final_guardian",
  "worldId": "world5",
  "type": "ending",
  "title": "最终结局：守护者",
  "dialogue": "你成为了守护者...\n\n【最终章 守护者结局】",
  "background": "ch05-void",
  "characters": ["creator"],
  "nextSceneId": null
}
```

### 方案 2: 修复回退逻辑

修改 `loadScene` 函数，当场景不存在时：
- 不跳转到第一个场景
- 显示错误提示
- 或者跳转到章节的最后一个已知场景

```javascript
if (!scene) {
  console.error('场景不存在:', sceneId)
  
  // ❌ 不要回退到第一个场景
  // const firstScene = getLoadedScript()?.scenes?.[0]
  
  // ✅ 显示提示并保持在当前场景
  alert('场景加载失败：' + sceneId + '\n请报告此问题。')
  
  // 或者跳转到标题页
  // window.router?.navigateTo('title')
  
  return
}
```

### 方案 3: 跨章节场景共享

创建一个全局结局场景文件，包含所有章节可能引用的结局：

```
src/data/script_endings.json
{
  "scenes": [
    {"id": "ending_destroyer", ...},
    {"id": "ending_guardian", ...},
    {"id": "ending_final_*", ...}
  ]
}
```

然后在 `Game.vue` 中加载所有结局场景。

---

## 推荐执行顺序

1. **立即修复**：修改 `loadScene` 的回退逻辑，避免跳到第一章
2. **短期修复**：在第 5 章添加所有缺失的最终结局场景
3. **长期修复**：整理所有章节的场景引用，确保所有 `nextSceneId` 都指向存在的场景

## 诊断命令

```bash
# 检查所有无效的场景引用
cd /home/admin/openclaw/workspace/rift-oath
node -e "
const fs = require('fs');
const files = ['script_ch1_full.json', 'script_ch2.json', 'script_ch3.json', 'script_ch4.json', 'script_ch5.json'];
const allScenes = new Set();
const allChoices = [];

files.forEach(file => {
  const data = JSON.parse(fs.readFileSync('src/data/' + file, 'utf-8'));
  data.scenes.forEach(s => {
    allScenes.add(s.id);
    if (s.choices) {
      s.choices.forEach(c => {
        allChoices.push({from: s.id, choice: c.id, next: c.nextSceneId, file});
      });
    }
    if (s.nextSceneId) {
      allChoices.push({from: s.id, choice: null, next: s.nextSceneId, file});
    }
  });
});

const invalid = allChoices.filter(c => c.next && !allScenes.has(c.next));
console.log('无效跳转目标 (' + invalid.length + '个):');
invalid.forEach(c => console.log(c.from + ' → ' + c.next + ' (不存在) [' + c.file + ']'));
"
```
