## 🤔 可能的问题

### 假设 A: 路径前缀问题
如果 Console 显示的 URL 是：
```
https://kidsmartghost.github.io/images/...  (缺少 /rift-oath/)
```
→ **原因**: 代码中的 `import.meta.env.BASE_URL` 没有正确获取

**解决方案**: 硬编码 BASE_PATH

### 假设 B: 缓存问题
如果直接访问图片 URL 能看到，但游戏中不显示
→ **原因**: 浏览器缓存了旧版本的 JS 文件

**解决方案**: 强制刷新 (Ctrl+Shift+R) 或清除缓存

### 假设 C: GitHub Pages 未更新
如果直接访问图片 URL 是 404
→ **原因**: GitHub Actions 部署失败或还未完成

**解决方案**: 检查 Actions 状态，重新触发部署

## 📋 当前代码状态

const BASE_PATH = import.meta.env.BASE_URL || '/rift-oath/'

function imagePath(path) {
  const cleanPath = path.replace(/^[/\.]+/, '')
  return BASE_PATH + cleanPath
}

// 使用示例
image: imagePath('images/backgrounds/ch01-church.png')
// 返回："/rift-oath/images/backgrounds/ch01-church.png"
