# 404 错误修复指南

**问题**: `Failed to load resource: ch01-church.png:1 (404 Not Found)`

---

## 🔍 原因分析

可能的原因：

1. **GitHub Pages 部署未完成** - 最新代码还在部署中
2. **浏览器缓存** - 浏览器缓存了旧版本的文件
3. **CDN 缓存** - GitHub 的 CDN 还没更新

---

## ✅ 解决方案

### 方案 1: 检查部署状态（最重要）

1. 访问：https://github.com/kidsmartghost/rift-oath/actions
2. 查看最新的 workflow 状态
3. 等待绿色 ✅ 标记出现

**部署通常需要 1-3 分钟**

---

### 方案 2: 强制刷新浏览器

**Chrome/Edge:**
```
Ctrl + Shift + R  (Windows/Linux)
Cmd + Shift + R   (Mac)
```

**或者清空缓存:**
1. 按 F12 打开开发者工具
2. 右键点击刷新按钮
3. 选择「清空缓存并硬性重新加载」

---

### 方案 3: 添加版本号参数（代码层面）

如果问题持续，可以在图片路径后添加版本参数绕过缓存：

```javascript
// src/data/assets.js
backgrounds: {
  'ch01-church': {
    id: 'ch01-church',
    world: 'world1',
    image: '/images/backgrounds/ch01-church.png?v=20260318'  // 添加版本号
  }
}
```

---

### 方案 4: 检查文件路径大小写

GitHub 使用 Linux 系统，**区分大小写**：

```bash
# 检查实际文件名
ls public/images/backgrounds/

# 应该看到:
# ch01-church.png  ✅ 正确
# CH01-CHURCH.png  ❌ 错误
```

---

##  验证步骤

### 1. 检查 GitHub Pages 是否部署完成

访问：https://github.com/kidsmartghost/rift-oath/commits/main

查看最新提交的时间，确保已经部署。

### 2. 检查图片是否可访问

直接访问图片 URL：
```
https://kidsmartghost.github.io/rift-oath/images/backgrounds/ch01-church.png
```

如果显示图片 → 部署成功，是缓存问题  
如果 404 → 部署还未完成或有其他问题

### 3. 查看浏览器 Console

按 F12 → Console 标签，查看完整错误信息。
